# 📊 Metrics Tracking — Phase 0 PMF 검증 데이터 표

> **목적**: 인터뷰 5명 + 베타 30명 데이터를 한 곳에 합산해 **Go/No-Go** 결정에 사용한다.
> **저장**: 본 MD는 템플릿. 실제 데이터는 Notion 또는 Google Sheets에 옮겨 운영 (라이브 업데이트 위해).

## 🎯 Go/No-Go 게이트 (재확인)

```
┌─────────────────────────────────────────────────────────┐
│  ✅ GO   ───  결제 의향 ≥ 30% AND "낫다" 답변 ≥ 50%      │
│  ❌ NO-GO ──  미만 시 → 포지셔닝 재정의 / B2B 피벗 검토   │
└─────────────────────────────────────────────────────────┘
```

## 3️⃣ Go/No-Go 결정표

| 지표 | 기준 | 실측값 | 통과? |
|---|---|---|---|
| 결제 의향 | ≥ 30% | __% | ✅/❌ |
| "우리 게 낫다" | ≥ 50% | __% | ✅/❌ |
| NPS | ≥ 0 | __ | ✅/❌ |
| 베타 30명 → 영상 생성 전환 | ≥ 60% | __% | ✅/❌ |
| 베타 7일 재방문 | ≥ 40% | __% | ✅/❌ |

### 결정 매트릭스

```
        결제 의향 ≥ 30%        결제 의향 < 30%
                ↓                    ↓
"낫다" ≥ 50%   ✅ GO Phase 1       ⚠️ 가격 재정의
                                   (도구 좋지만 호가 낮음 → 무료+광고/마켓플레이스 모델)

"낫다" < 50%   ⚠️ 차별점 재정의    ❌ 피벗
                                   (B2B 화이트레이블 / 교육 패키지 검토)
```

## ⚠️ 데이터 수집 시 주의

1. **개인정보 분리 보관**: 인터뷰 노트에서 이름·이메일은 별도 파일에. 분석 노트에는 ID만.
2. **녹화/녹음 동의 필수**: 동의 없으면 노트만 작성.
3. **응답 강요 금지**: 매출·소득 답변 어려워하면 패스. 강요 시 답변 신뢰도 ↓.
4. **확증 편향 경계**: "우리 가설이 맞는 답변"만 기억하지 말 것. 매주 회의에서 *반대 신호* 1개씩 공유.
