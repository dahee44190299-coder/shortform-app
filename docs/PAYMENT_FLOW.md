# 💳 결제 + 마케팅 자동화 흐름

> 사용자가 가입 → 무료 사용 → Pro 전환 → 매출 발생 → 자동 마케팅까지 전 과정.

---

## 1. 결제 흐름 (Pro 업그레이드)

### 한국 결제 (PortOne)

```
┌──────────────┐      ┌───────────────────┐      ┌──────────────┐
│ 사용자 (Free) │      │ Streamlit App     │      │ PortOne      │
└──────┬───────┘      └─────────┬─────────┘      └──────┬───────┘
       │                        │                         │
       │ "Pro 업그레이드" 클릭    │                         │
       ├───────────────────────>│                         │
       │                        │ create_payment_request  │
       │                        ├────────────────────────>│
       │                        │                         │
       │                        │  결제 위젯 URL          │
       │                        │<────────────────────────┤
       │                        │                         │
       │ 결제 위젯 (카카오/카드)  │                         │
       │<───────────────────────┤                         │
       │                        │                         │
       │  결제 진행              │                         │
       ├────────────────────────────────────────────────>│
       │                        │                         │
       │                        │   Webhook (결제 완료)   │
       │                        │<────────────────────────┤
       │                        │                         │
       │                        │ 1. 서명 검증            │
       │                        │ 2. paid_pro.json 추가   │
       │                        │ 3. user에게 알림        │
       │  ✅ Pro 활성화          │                         │
       │<───────────────────────┤                         │
       │                        │                         │
```

**구현 단계** (현재 미구현, Phase 5 예정):
1. `payment.py` 모듈 생성
2. PortOne SDK 통합 (`pip install iamport-rest-client-python`)
3. Streamlit 앱에 결제 위젯 컴포넌트
4. Webhook 엔드포인트 (FastAPI 별도 서비스)
5. `paid_pro.json` (또는 SQLite) 사용자 등록

**현재는**: Free 영구 무료 + Founder/Invitee 영구 무료 → **결제 시스템 없이도 첫 사용자 모집 가능**.

### 글로벌 결제 (Stripe) — 나중에

PortOne 대신 Stripe만 사용:
- 신용카드 직결 (Visa/Master)
- 자동 부가세 계산 (EU VAT, US sales tax)
- Subscriptions API (월/연 자동 결제)

## 3. NHN 마케팅 자동화 5대 전략 적용 매핑

본 프로젝트가 [NHN 글](https://inside.nhn.com/tech/204)의 5대 전략을 어떻게 구현하는지:

| NHN 전략 | 본 프로젝트 구현 |
|---|---|
| AI 기반 추천 | ✅ regeneration.py (저성과 영상 자동 다른 Hook 추천) |
| 고객 데이터 플랫폼 (CDP) | ✅ tracking + eval_metrics + pmf_survey 통합 (사용자별 360도 데이터) |
| 대화형 마케팅 (챗봇) | ⏳ Phase 5 — 카카오톡 알림톡 자동 발송 |
| 옴니채널 통합 | ⏳ Phase 5 — Streamlit + Vercel + 카카오 통합 |
| 영상 콘텐츠 자동화 | ✅ 본 프로젝트의 핵심 |

**한국 특화 추가** (Phase 5 예정):
- 카카오톡 알림톡: 매출 발생 시 자동 축하 메시지
- 네이버 광고: 검색 키워드 "쿠팡 파트너스 영상" 매입
- 디스코드 봇: 인디해커 한국 커뮤니티 자동 알림

## 4. 결제 없이도 매출 나오는 서비스들

### 패턴 A — **GitHub Sponsors / Patreon / Ko-fi**
- 도메인 X, 결제 처리 X, 사업자 등록 X
- 사용자가 **개발자 프로필**에 직접 후원
- 본 프로젝트도 가능: `https://github.com/sponsors/dahee44190299-coder`

### 패턴 B — **affiliate revenue share (본 프로젝트의 기본 모델)**
- 도메인 X, 자체 결제 X
- 쿠팡 파트너스 referrer subId만 부착하면 쿠팡이 우리 매출 송금
- 본인 사업자등록 + 쿠팡 파트너스 정산 계좌만 있으면 됨

### 패턴 C — **GitHub Marketplace / Streamlit Cloud Apps**
- 본인 도메인 X
- 플랫폼이 결제 처리 + 수수료 (15-30%) 떼고 송금
- 본 프로젝트 가능: GitHub Apps 마켓플레이스 등록

### 패턴 D — **유튜브/블로그 콘텐츠 → 후원/광고**
- "이 도구로 월 50만원 벌었어요" 영상 → 광고 수익
- 도메인 X, 본인 채널만 있으면 됨

### 결론
**도메인 없이도 매출 가능. 본 프로젝트는 패턴 B(affiliate revenue share) + D(콘텐츠 마케팅)로 시작하는 게 가장 빠름.**

도메인은 **첫 매출 발생 후 신뢰도 향상용**으로 사도 충분합니다 (₩15,000/년).

## 5. 첫 매출까지 가장 빠른 경로

### Week 1
1. Streamlit Cloud 배포 (`https://your-app.streamlit.app`)
2. 본인이 첫 사용자 → 영상 5개 직접 생성 → 본인 SNS 업로드
3. **본인 쿠팡 파트너스 계정에 매출 발생 시작**

### Week 2
1. GitHub Sponsors 활성화 (백업 수익 채널)
2. 케이스 스터디 1건 작성 (본인 매출 데이터 공개)
3. X/디스코드/노마드코더 카페에 "오픈소스로 만들었어요" 글

### Week 3
1. 무료 가입자 5-10명
2. 그중 매출 발생자 1-2명 → 자동 케이스 스터디 발행
3. SEO 인덱싱 시작 (Vercel 랜딩 페이지)

### Week 4
1. PMF 설문 결과 수집 (in-product)
2. **Pro 가격 검증** (만약 결제 의향 30% 이상 → 다음 달 Pro 출시)
3. 매출 첫 ₩10,000 도달

→ **첫 달 안에 매출 발생 가능**, 도메인은 그 후에 사도 됨.
