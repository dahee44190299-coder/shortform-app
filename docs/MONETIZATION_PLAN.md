# 💰 수익화 자동화 계획

> 외부 비판 검토 결과 (2026-04-27): "유료 사용자 첫 10명 모집 계획이 없다" + "타겟 페르소나가 가난하다"
> 대응: **Revenue Share 기본 모델** — 사용자가 매출 내야 우리도 돈 번다.

---

## 핵심 원칙

1. **무료 = 모든 기능** (고정 비용 부담 0)
2. **수익은 사용자 매출의 N% 자동 회수** (성과 기반)
3. **유료 플랜은 "본인 affiliate ID 사용" + 부가 자동화** (브랜딩/배치 업로드)

→ 가난한 타겟 페르소나도 **선결제 없이** 시작 가능 + 우리는 **사용자 매출과 함께 성장**

## Tier 구조

### 🆓 Free (영구)
| 기능 | 무료 |
|---|---|
| AI 대본 생성 (LLM judge 자동 재생성 포함) | ✅ |
| 카테고리 가이드 + Hook 추천 | ✅ |
| 영상 자동 편집 (Pexels + 자막 + TTS) | ✅ |
| 추적 링크 자동 생성 (subId) | ✅ |
| **단**: deeplink subId에 본 도구 referrer 코드 부착 | ⚠️ |
| 매출 추적 대시보드 | ✅ |

**수익화 메커니즘**:
- 사용자가 만든 영상의 추적 링크 = `subId=vid_xxx_REFERRAL_shortformapp`
- 쿠팡이 인식하는 referrer는 **사용자 본인** (수수료는 사용자가 받음)
- 단, 본 도구가 **별도 추적 채널을 통해 부가 수익 회수**:
  - 옵션 A: 본 도구 운영자가 별도 쿠팡 파트너스 계정으로 **케이스 스터디 페이지에서 동일 상품 cross-promotion**
  - 옵션 B: **유료 전환 시 시도 시점부터 referrer 제거**

### 💎 Pro ($9/월 또는 매출 5% revenue share)
| 추가 기능 |
|---|
| referrer 코드 제거 (100% 본인 매출) |
| 자동 업로드 (YouTube/TikTok API) |
| 배치 영상 생성 (10개 동시) |
| 우선 모델 (Sonnet 4.5 → Opus) |
| LLM judge 합격선 사용자 정의 (기본 80 → 90) |
| 케이스 스터디 자동 발행 (자체 도메인) |
| 멀티 affiliate (쿠팡 + Amazon + TikTok Shop) |

**선택형 결제**:
- 정액제 $9/월 = 안정적 사용자 (매출 무관)
- Revenue share 5% = 매출 기반 (월 매출 $200+ 사용자에게 정액보다 비쌈, 매출 적은 사용자에게 유리)

## 첫 10명 유료 사용자 모집 — 구체적 계획

### Phase A — Build in Public (Week 1-4)
1. **본인이 첫 사용자**: 본 도구로 쿠팡 파트너스 영상 30개 자동 생성
2. 매주 GitHub README + 블로그에 진척 공개:
   - 영상 N개 생성 → 클릭 X회 → 매출 Y원
   - LLM judge 평균 점수 추이
   - 카테고리별 매출 비교
3. **자동화**: `marketing_auto.py`가 매주 일요일 stats 자동 게시

### Phase B — 케이스 스터디 SEO (Week 5-12)
1. 매출 발생한 영상 = 자동 케이스 스터디 페이지로 변환
2. URL 구조: `shortform-app.dev/case/{slug}` (정적 사이트, GitHub Pages)
3. 타겟 키워드:
   - "쿠팡 파트너스 영상 만드는 법"
   - "쿠팡 파트너스 매출 후기"
   - "쇼츠 자동화 도구"
4. **자동화**: 새 매출 발생 시 케이스 스터디 자동 PR 생성 → main 머지 시 자동 deploy

### Phase C — Inbound Conversion (Week 13+)
1. 무료 사용자가 매출 데이터 입력하면:
   - 본인 케이스 스터디 페이지 자동 생성 (옵트인)
   - "이 영상으로 X원 벌었어요" 소셜 카드 자동 생성
2. 케이스 스터디 페이지마다 본 도구 cross-link → 무료 가입 유도
3. 무료 → Pro 전환 트리거:
   - 월 매출 $50 도달 시 자동 노티: "5% revenue share면 본인 100% 매출 확보 가능"
   - 영상 50개 도달 시 자동 노티: "배치 업로드로 시간 80% 절약"

### Phase D — In-product PMF Survey (즉시)
- 모든 사용자에게 **"이 기능 유료라면 결제하시겠어요?"** 마이크로 설문 (월 1회)
- NPS 자동 수집 (10점 만점)
- 결제 의향 30% 이상 도달 → 유료 플랜 정식 출시

## 위험 시나리오 + 대응

| 위험 | 확률 | 대응 |
|---|---|---|
| 쿠팡이 referrer subId 사용 차단 | 30% | Pro 플랜 매력 강화 (자동 업로드 등) |
| 무료 사용자 매출 0인 채로 이탈 | 70% | LLM judge로 품질 보장 → 매출 발생 가능성 ↑ |
| 케이스 스터디 SEO가 안 떠서 유입 0 | 50% | 본인 영상 직접 발행 (백업 채널) |
| 결제 의향 < 30% | 40% | B2B 피벗 (MCN 화이트레이블, 강의 컨버전) |
